Mr.Market EVM Order Funding Router Architecture

v1 decision diagram: instance-bound router, server-signed order authorization, immediate forwarding to Mr.Market controlled fundingReceiver, ledger-first accounting.

Contract v1 Scope

  • • One router contract binds one Mr.Market instance.
  • • Verifies server/admin signed OrderAuthorization.
  • • Receives user funds and immediately forwards to fundingReceiver.
  • • Emits OrderFunded; does not maintain order balances.
  • • Keeps emergency pause/unpause and admin/receiver rotation.

Ledger Boundary

  • • Ledger remains the balance source of truth.
  • • OrderFunded confirmations create typed deposit_credit.
  • • Balance is scoped by orderId + asset, never by wallet alone.
  • • idempotency key: chainId + txHash + logIndex.
  • • Any mismatch enters manual_review, not generic adjustment.

Server Responsibilities

  • • SIWE login, campaign/strategy validation, and EIP-712 signing.
  • • Store orderRef → orderId authorization records.
  • • Index chain events and wait required confirmations.
  • • Advance MarketMakingOrder state after ledger credit.
  • • Handle later CEX top-up, withdrawal, reconciliation, and audit.
